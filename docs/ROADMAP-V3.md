# 司南基金 V3 路线图

V1（M0–M5）+ V2（V2-0~V2-7）已全部上线。V3 往「更准、更深」走：补数据地基、把已有数据
变成可分享的结论、深化资产分析。仍坚持「个人辅助、非机构、非自动交易」。

> 约束：移植自蜉蝣基金（FundVal）的功能一律在司南前端用 TS **全新复刻，不修改 FundVal 任何文件**
> （沿用 V2-4 盘中估值的做法）。

按「价值 ÷ 工作量」排序，逐阶段交付、可独立验收。

---

## V3-0 · 指数行情条（移植自 FundVal）

首页顶部横向滚动条：纳斯达克 / 标普500 / AU9999 金价 / 上证 / 沪深300 实时数值。
**纯前端 JSONP**：`qt.gtimg.cn/q={codes}`（window.v_*）+ 新浪 `hq.sinajs.cn/list=AU0`（金价）。
A 股休市显示上一交易日收盘；30s 刷新；离线回退缓存。不依赖后端。
**工作量**：小。 **验收**：交易时段实时滚动，休市/离线优雅降级。

## V3-1 · 基金体检报告

把评分 / 信号 / 回测 / 盘中估值 / 经理 / 收益整合成一页流「体检报告」，可一键分享 / 导出图片。
复用现有数据，零新接口。
**工作量**：小。 **验收**：报告完整、口径中立、附免责声明。

## V3-2 · 十大重仓股（移植自 FundVal）

详情页展开基金十大重仓股 + 个股涨跌幅。`fundf10.eastmoney.com/FundArchivesDatas.aspx?type=jjcc`
（返回 `var apidata={content:"HTML"}`，需解析 HTML）+ push2 `ulist.np` 补个股涨跌幅。
是「持仓穿透」的单只基金版、第一块拼图。
**工作量**：中。 **验收**：重仓股列表 + 占比 + 涨跌正确。

## V3-3 · 持仓穿透（组合级）

把自选/持仓里每只基金的十大重仓股聚合 → 组合**真实个股暴露**、**行业分布**、个股集中度
（同一只股票被多只基金重仓时合并）。建立在 V3-2 之上。
**工作量**：中高。 **验收**：穿透后的个股/行业占比与各基金权重一致。

## V3-4 · 富集筛选

后端定时富集任务把热门基金详情入库，`/api/funds` 支持按**规模 / 费率 / 近N年收益 / 最大回撤 /
夏普 / 经理任期**筛选 + 排序。让「选基」从「知代码才查」变成真能淘。
**依赖**：后端富集任务。 **工作量**：中高。 **验收**：多维筛选/排序结果正确、性能可接受。

## V3-5 · 真实 PE/PB 估值

指数基金映射到所跟踪指数（沪深300/中证500/中证白酒…），取**指数真实 PE/PB 历史分位**替代
现在的「净值分位代理」，喂给择时估值层。直接修掉强趋势长牛被误判高估的命门。
**依赖**：指数估值数据源 + 基金↔指数映射表。 **工作量**：高。
**验收**：指数基金估值层用真实分位、回测踏空问题缓解；非指数基金优雅降级。

## V3-6 · 自然语言选基

「近三年年化>15%、回撤<20%、规模>50亿的混合基金」→ LLM 解析成筛选条件查 V3-4 的库。
**依赖**：V3-4 + Anthropic key（复用 V2-5 的 key 开关）。 **工作量**：中。

## V3-7 · 相似 / 同类更优推荐

看一只基金时，推同类里评分更高、费率更低、回撤更小的替代。**依赖**：V3-4 富集库。 **工作量**：中。

## V3-8 · 收益归因

资产页：今日 / 本月收益拆解到每只基金、每个账户的贡献度。复用盘中估值。 **工作量**：中。

## V3-9 · 多源容灾

天天基金主源外接 AKShare / 新浪备源，主源失败自动降级，避免单点全挂。 **工作量**：中。

## V3-10 · 组合历史净值曲线

每日存一次组合总资产快照，画资产随时间变化曲线。 **依赖**：快照持久化（localStorage/Gist）。 **工作量**：中。

## V3-11 · 体验工程

暗黑模式（Vant 暗色主题）+ 数据导出（Excel/CSV）+ 下拉刷新 / 骨架屏。顺手提质感。 **工作量**：小。

## V3-12 · 同一基金跨多账户

唯一键由 `code` 改 `账户|code` 复合键，让同一基金能在多账户分别记。
**关键约束**：迁移必须**向后兼容**——加 `schema` 版本号；account 为空时复合键退化成 code；
新旧读写都能解析，避免新旧版本 App 同写一个 Gist 时错乱。token / 权限不受影响。
**工作量**：中（数据模型 + Gist 合并逻辑重构）。 **验收**：跨账户同基金独立记账、老数据不丢、多设备不冲突。

## V3-13 · 大类资产记账

把现金 / 股票 / 黄金也纳入，做大类资产配置总览。最偏记账、离核心最远，放最后。 **工作量**：大。

V3-0 → V3-1 → V3-2 → V3-3 → V3-4 → V3-5 → V3-6 → V3-7 → V3-8 → V3-9 → V3-10 → V3-11 → V3-12 → V3-13

决策智能（NL选基/同类更优）依赖筛选库排其后，资产/容灾/体验/跨账户收尾。
