# 司南基金 V2 路线图

V1（M0–M5）已上线。V2 在其上做增强，仍坚持「个人辅助、非机构、非自动交易」的定位。
按「价值 ÷ 工作量」排序，逐阶段交付、可独立验收。

> 原始 V2 候选（需求规格）：回测、组合分析、定投计算器、消息推送、AI 解读、多账户。

---

## V2-0 · 自选云同步（先做）

**痛点**：Render 免费档 SQLite 是临时盘，后端一重新部署自选就清空。
**方案**：参考蜉蝣基金（FundVal）做法 —— 自选改为**前端 localStorage 存储 + GitHub Gist 云同步**，与后端解耦。

- 自选数据（含时间戳 + 删除墓碑）存浏览器 localStorage；
- 用 GitHub Personal Access Token（gist 权限）把数据同步到一个私有 Gist；
- 自动：变更防抖推送、加载/切回页面时拉取；按 code + updated_at 合并（新者胜），删除用墓碑（保留 30 天）；
- 自选页加「云同步」设置：填 Token、上传/下载/清除、同步状态。

**收益**：后端怎么重部署都不丢自选；多设备可同步。
**验收**：删后端重部署，自选仍在；两台设备/浏览器配同一 Token 后自选一致。

## V2-1 · 定投计算器

选基金 + 月投额 + 期限，用历史净值回放，算定投收益 vs 一次性买入，给曲线与年化。
**依赖**：现有净值数据。 **工作量**：小。 **验收**：结果与手算一致，曲线合理。

## V2-2 · 组合分析

把「自选」升级为「持仓」（每只加份额/金额）→ 组合市值、按类型配置、合并收益、
风险与分散度（相关性）。
**依赖**：V2-0 的本地存储结构扩展（加金额字段）。 **工作量**：中。
**验收**：组合层指标正确，配置/收益/风险一目了然。

## V2-3 · 择时回测

按 V1 的三层信号在历史上模拟操作（买入/定投/持有/减仓），统计收益、最大回撤、
胜率，并与「一直持有」基准对比，验证策略有效性。
**依赖**：timing 模块。 **工作量**：中高。 **验收**：回测可复现、与基准对比清晰、每笔可追溯。

## V2-4 · AI 基金解读

用 Claude API 把评分/信号/指标合成一段自然语言点评与操作建议（个人参考，非投资建议）。
**依赖**：Anthropic API key（走费用）。 **工作量**：中。
**验收**：解读贴合数据、口径中立、附免责声明。

## V2-5 · 信号推送（兼解决保活）

定时任务每隔几分钟算自选信号，信号变化时推送（Bark / Server酱 / 邮件）；
该定时任务顺带每隔几分钟 ping 后端，**免去 Render 冷启动**。
**依赖**：定时任务（GitHub Actions cron 或后端调度）+ 推送通道。 **工作量**：中高。
**验收**：信号变化能收到推送；后端不再休眠。

## V2-6 · 多账户资产管理

跨账户记录真实持仓、汇总资产。最重、最贴近记账，放最后。
**工作量**：大。 **验收**：多账户汇总正确。
